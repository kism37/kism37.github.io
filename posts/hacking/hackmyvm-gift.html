Gift

When the web has nothing to give, go knock on SSH's door.

Platform HackMyVM
Read 4 min
Difficulty Easy
Tags SSH · Brute Force · Recon

Overview

Gift is an easy HackMyVM machine that teaches a clean lesson: when one door is locked, check the others. HTTP gave us nothing. SSH gave us everything. The vulnerability here isn't exotic — it's a weak root password on an exposed SSH port, brute-forceable with the right wordlist.

ℹ All testing was done in a local lab environment on HackMyVM. Target IP: 192.168.8.179, added to /etc/hosts as gift.hmv.

Reconnaissance

Started with an fping sweep to identify the target on the local network. With two known addresses already taken (AP and my machine), the remaining IP was the target. 

# Identify live hosts
fping -a -g 192.168.8.0/24 2>/dev/null

# Full port scan
nmap -sC -sV -p- 192.168.8.179

Nmap returned two open ports: 22 (SSH) and 80 (HTTP). Nothing unusual in the service versions. Added the target to /etc/hosts and moved on.

Enumeration

Hit port 80 first. Ran dirb and nikto against the HTTP service looking for hidden directories or misconfigurations. Found nothing beyond a default index.html. The web surface was completely flat — no forms, no parameters, no interesting headers.

⚠ When HTTP is a dead end, SSH becomes the priority target. Don't spend too long banging on a door that has nothing behind it.

With HTTP exhausted, I pivoted to port 22. The attack surface here is straightforward: SSH with no key-based auth enforcement means password brute-force is viable if you have the right wordlist and a reasonable user list.

Exploitation

01

Build user wordlist

Created a minimal user list with two candidates: gift (machine name, often a hint) and root.

02

Brute force SSH with ncrack

Ran ncrack against port 22 with the user list and rockyou.txt as the password wordlist.

03

Hit confirmed

Got a valid credential hit almost immediately.

04

SSH in as root

Logged in directly as root. No privilege escalation needed — straight to both flags.

ncrack -U users.txt -P /usr/share/wordlists/rockyou.txt ssh://192.168.8.179
⚑ Credential found: root:simple — SSH'd in as root, collected user.txt and root.txt.

Takeaways

Gift is a machine about not getting tunnel vision. The instinct when you hit a wall on HTTP is to dig harder — more fuzzing, more tools, more time. Sometimes the right move is to just turn around and look at what else is open.

The real finding here is a weak root password on an internet-facing SSH port. In a real engagement that's a critical — direct root access with no exploitation, just a dictionary word. The fix is obvious: disable password auth, enforce key-based SSH, never expose SSH directly if you can help it.

Machine name as username hint is a pattern worth filing away. CTF and lab designers love to hide usernames in plain sight.
